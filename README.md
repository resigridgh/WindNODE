# WindNODE: Wind Turbine Power Prediction using Neural ODE

## 📌 Project Overview

This project focuses on predicting **wind turbine power generation** using a real-world Supervisory Control and Data Acquisition (SCADA) dataset. The goal is to model the nonlinear dynamics of wind energy systems and develop accurate data-driven prediction models using **Neural Ordinary Differential Equations (Neural ODEs)**.

## 📊 Dataset Description

### 🔗 Dataset Source

* Name: Wind Turbine SCADA Dataset
* Link: [https://www.kaggle.com/code/akdagmelih/wind-turbine-power-prediction-gbtregressor-pyspark/input](https://www.kaggle.com/code/akdagmelih/wind-turbine-power-prediction-gbtregressor-pyspark/input)
* Original Source: Public SCADA dataset available on Kaggle (collected from real wind turbine operation)

### 📦 Dataset Size

* Total Samples: 50,530 observations
* Sampling Interval: 10 minutes

### ⚙️ What the Dataset is About

This dataset contains SCADA data from a wind turbine.

It records operational parameters such as wind conditions and generated power over time. These measurements are commonly used in industry for monitoring, control, and predictive analytics.


### 📈 Features (Input Variables)

* Date/Time: Timestamp recorded at 10-minute intervals
* Wind Speed (m/s): Wind speed at turbine hub height
* Wind Direction (°): Direction of incoming wind
* Theoretical Power Curve (kWh): Manufacturer-provided expected power output
* LV ActivePower (kW): The power generated by the turbine for a certain moment


### 🎯 Label (Target Variable)

* LV ActivePower (kW): Actual electrical power generated by the wind turbine

This is a regression problem, where the goal is to predict continuous time power output.


## 🧠 Problem Statement

The main objective of this project is:

To accurately predict wind turbine power generation using Neural Ordinary Differential Equations (Neural ODEs).


## 🤖 Proposed Approach

* Preprocess time-series SCADA data
* Extract temporal features (month, hour)
* Train a Neural ODE model to learn system dynamics
* Compare predictions with actual power output
